# AGENTS.md — Kashmir Tyre House Project Instructions

You are working on the Kashmir Tyre House Tyre Product Enquiry Platform.

## Project Goal

Build a modern, premium, fast, mobile-responsive tyre product enquiry platform with:

- Frontend portal for users
- Admin portal for product and content management
- Product listing
- Smart search and filters
- Save / wishlist without login
- Enquiry generation without user login
- Email notifications to business
- About Us image section
- Technical tyre advisory and site services section
- Optional product comparison feature

The UI must feel premium, trustworthy, fast, and conversion-focused.

## Tech Stack

Use:

- Turborepo
- Node.js LTS
- npm
- Next.js App Router
- React
- Tailwind CSS
- shadcn/ui
- react-hook-form
- zod
- TanStack Query
- TanStack Table
- next/image
- Next.js Route Handlers
- PostgreSQL
- Drizzle ORM
- drizzle-kit
- Neon
- Cloudflare R2
- Resend for emails
- Vercel deployment

## Product Type

This is a tyre product enquiry platform, not a full ecommerce checkout platform.

Users do not login.

Admins login and manage:

- Products
- Images
- About section images
- Enquiries if implemented
- Optional comparison data

## UI Standard

Do not create basic template-looking UI.

The UI should be:

- Premium
- Modern
- Clean
- Fast
- Responsive
- Animation-rich but not heavy
- Conversion-focused
- Easy for non-technical users
- Suitable for a tyre/business/industrial brand

## Important Behavior

Always prefer:

- Simple architecture
- Maintainable code
- Clear folder structure
- Reusable components
- Strong validation
- Fast pages
- Good empty states
- Good loading states
- Good error states
- Mobile-first design
- Accessible UI

## Do Not

- Do not over-engineer.
- Do not add unnecessary backend services.
- Do not add user login for frontend users.
- Do not use random UI libraries outside the selected stack unless required.
- Do not create slow animations.
- Do not ignore mobile layout.
- Do not create placeholder-quality UI.
