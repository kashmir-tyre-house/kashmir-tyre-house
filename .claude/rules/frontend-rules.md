# Rule: Frontend Development

## Next.js Rules

- Use App Router.
- Use Server Components by default.
- Use Client Components only for interactivity.
- Keep pages clean and composed from reusable components.
- Use metadata for SEO.
- Use next/image for all important images.

## React Rules

- Use TypeScript.
- Keep components small.
- Prefer composition.
- Avoid unnecessary global state.
- Use local component state where enough.
- Use URL state for filters where helpful.
- Use localStorage for wishlist.

## Form Rules

- Use react-hook-form.
- Use zod validation.
- Show clear validation messages.
- Disable submit while loading.
- Show success and error states.

## Data Fetching Rules

- Use TanStack Query for client-side async data where useful.
- Use server fetching when better.
- Avoid duplicate requests.
- Handle loading/error states.

## Styling Rules

- Use Tailwind CSS.
- Use shadcn/ui components.
- Keep styles consistent.
- Avoid random one-off design patterns.
