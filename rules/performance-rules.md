# Rule: Performance

The app must feel fast.

## Required Practices

- Use Server Components by default.
- Keep client JavaScript minimal.
- Use next/image.
- Use responsive image sizes.
- Lazy-load non-critical sections.
- Use skeleton loading states.
- Avoid large dependencies.
- Avoid unnecessary re-renders.
- Keep animations lightweight.
- Avoid layout shift.
- Cache where appropriate.

## Product Listing

- Do not load unnecessarily huge images.
- Use pagination or lazy loading if product count grows.
- Keep filtering fast.
- Use clean empty states.

## Admin

- Use pagination for large tables.
- Avoid loading all heavy records at once.
- Keep admin forms responsive and fast.
