# Skill: Create Responsive Component

Use this skill when creating reusable UI components.

## Goal

Create components that are clean, reusable, accessible, responsive, and visually polished.

## Steps

1. Define component purpose.
2. Define props with TypeScript.
3. Build mobile-first layout.
4. Add responsive variants.
5. Add accessible labels and keyboard behavior where needed.
6. Add empty/loading states if needed.
7. Add subtle hover/focus states.
8. Keep component reusable and not tightly coupled.

## Component Rules

- Use TypeScript.
- Use Tailwind CSS.
- Use shadcn/ui primitives where suitable.
- Avoid hardcoded data.
- Keep props clear.
- Avoid unnecessary client-side state.
- Use composition where useful.

## For Product Cards

Always consider:

- Product image
- Product name
- Brand
- Size
- Vehicle type
- Load index / ply rating
- Save button
- Add to enquiry button
- Compare checkbox if feature enabled
